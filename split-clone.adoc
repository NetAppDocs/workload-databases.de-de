= Einen Datenbankklon von der Quelldatenbank trennen
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
In NetApp Workload Factory für Datenbanken wird durch die Trennung eines Datenbankklons von seiner Quelldatenbank eine neue Datenbank erstellt, die eine bestimmte Menge an Speicherkapazität verbraucht.  Der Klon wird gelöscht, wenn die Aufteilung abgeschlossen ist und die neue Datenbank im Inventar angezeigt wird.

.Bevor Sie beginnen
Berücksichtigen Sie, wie viel Storage-Kapazität für die neue Datenbank benötigt wird. Falls erforderlich, link:https://docs.netapp.com/us-en/workload-fsx-ontap/increase-file-system-capacity.html["Erhöhen Sie die Kapazität des Filesystems"^] für das FSX für ONTAP-Dateisystem, bevor Sie beginnen.

.Schritte
. Melden Sie sich mit einem der link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["Konsolenerfahrungen"^]an.
. Wählen Sie das Menü aus image:workloads-menu-icon.png["Über das Hamburger-Menüsymbol kann man zu Workloads wie Speicher, EDA, KI, Datenbanken, VMware und Administration navigieren."] und wählen Sie dann *Datenbanken* aus.
. Wählen Sie im Menü „Datenbanken“ die Option „Sandboxen“ aus.
. Wählen Sie in Sandboxen das Aktionsmenü des Datenbankklons aus, den Sie aufteilen möchten.
. Wählen Sie *Split*.
. Wählen Sie im Dialogfeld „Teilen“ die Option „Teilen“ aus.
